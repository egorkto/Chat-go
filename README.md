# Go chat
<a id="description"></a>
Проект чата в реальном времени с аутентификацией на основе JWT-токенов. Проект разработан на языке Go. Реализована авторизация с использованием асимметричных RSA-ключей, их валидацией и обновлением. Общий чат реализован с ипользованием протокола Websocket

<a id="getting-started"></a>
## Начало работы

### Требования
* Установленный Go (версии 1.20 и выше)
* Docker и Docker Compose

### Пошаговая установка и локальный запуск

1. Склонируйте репозиторий с проектом:
   ```bash
   git clone https://github.com
   cd your-repo-name
   ```

2. Создайте файл окружения `.env` в корне проекта на основе [примера окружения](.env.example) из репозитория:

3. Сгенерируйте пару RSA-ключей для подписи и проверки JWT-токенов:
   ```bash
   make generate-keys
   ```

4. Запустите базу данных PostgreSQL в Docker-контейнере:
   ```bash
   make up-db
   ```

5. Примените миграции базы данных:
   ```bash
   make up-migration
   ```
   
6. Пробросить порты БД на локальную машину:
   ```bash
   make forward-port
   ```

7. Запустите проект локально:
   ```bash
   make run-chat
   ```

<a id="deploy"></a>
## Деплой

Проект разворачивается в Docker-окружении с помощью Docker Compose.

### Запуск приложения в Docker
Запустить контейнер БД и контейнер приложения
   ```bash
   make up-db
   make up-migration
   make up-chat
   ```
Эта команда соберет Docker-образ для `chat-app` и запустит его в фоновом режиме.

### Остановка сервисов
Чтобы остановить контейнер с чатом:
  ```bash
  make down-chat
  make down-db
  ```

<a id="project-workflow"></a>
## Работа с проектом
В [Makefile](Makefile) также указаны другие полезные команды:
* Полная очистки данных базы данных:
  ```bash
  make cleanup-db
  ```
  
* Создание миграции с названием `seq`:
  ```bash
  make create-migration seq=
  ```
  
* Очистка логов:
  ```bash
  make clean-logs
  ```
  
* Генерация swagger документации:
  ```bash
  make run-swag
  ```
